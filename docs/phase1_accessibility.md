Phase 1 Accessibility Baseline

- Keyboard navigation: All interactive controls reachable via Tab; visible focus rings; logical focus order.
- ARIA: Labels for charts, swatches, and sampling overlay. Descriptions for non-text visuals.
- Color contrast: Ensure primary text and UI elements meet WCAG AA contrast (4.5:1 for body text; higher for UI components).
- Images/Charts: Provide text alternatives; charts include axis labels and data descriptions.
- Resizable text: UI scales with user font-size preferences without breaking layout.
- Localization readiness: Strings prepared for i18n; RTL support considerations documented for Phase 2.
- Error handling: Accessible error messages with guidance to recover.
- Plan for Phase 2: Accessibility review and audit checklist with pass/fail criteria.
