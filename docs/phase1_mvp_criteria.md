Phase 1 MVP Criteria (Phase 2 readiness)

- Onboarding: Document the flow and content for a concise introduction to the app.
- Result presentation skeleton: Implement a compact ResultCard with key metric, confidence, interpretation line, and color swatches.
- Visualization placeholder: Include a simple histogram or basic color-space diagram with accessible labels and descriptions.
- Input sampling flow: Clear Start Sample action and basic error handling/state.
- Calibration concept: Basic tips or hint for calibration within the results flow; plan for Phase 2 calibration UI.
- Privacy snippet: A concise privacy statement on the main UI and an accessible link to more details.
- Accessibility baseline: Draft ARIA roles/labels for controls and visuals; keyboard navigation ready.
- Export/Share outline: Basic outline for exporting results in JSON/CSV and a screenshot/URL snippet for Phase 2.
- Design tokens: Document initial color/typography/spacing tokens for Phase 2 implementation.
- Phase 2 acceptance criteria: Clear pass/fail indicators for each MVP item and a checklist.
