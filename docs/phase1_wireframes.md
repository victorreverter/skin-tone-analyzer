Phase 1 Wireframes — Conceptual Explorations

Overview
- We present 3 directional concepts for Phase 2, each focusing on different user priorities.

Concept A — Clarity-first (Compact Results)
- Layout: Left column shows input area and a small set of controls; Right column shows ResultCard plus a compact SwatchPanel.
- ResultCard: shows key metric, confidence, short interpretation, and 2–3 swatches.
- Visuals: a slim histogram beneath the ResultCard; an OverlayToggle to show sampling points on the input image.
- Primary interactions: Start Sample, Calibrate hint, Export (later).

Concept B — Visual Analytics
- Layout: Left: large Color-Space visualization (CIELAB/CIELUV) with axes and a draggable slider; Right: detailed metrics and a larger histogram with tooltips.
- Overlay: sampling points overlay on the input image; calibration tips on the side.
- Primary interactions: sample, adjust view, calibrate, export.

Concept C — Guided Onboarding
- Layout: Step-by-step onboarding flow showing what is measured, how calibration works, and how to interpret results.
- Each step includes mini-tasks and contextual help; final step shows results with minimal visuals and a link to advanced analytics.

Recommended Direction (Phase 2 planning)
- After discussion, we should start with Concept A (Clarity-first) as MVP baseline for Phase 2, then iteratively extend with Concept B in a follow-up sprint.

Notes
- Wireframes are textual descriptions for Phase 2; low-fidelity visuals can be produced as needed.
